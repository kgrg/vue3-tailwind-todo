## Overview

TaskFlow is a single-page Vue 3 application for activity and task management. It uses a **feature-module architecture** where each domain (activities, auth, habits, labels, tasks) is a self-contained module with its own components, store, and types.

## Key Architectural Patterns

### 1. Feature-Module Pattern

Each domain lives in `src/modules/<feature>/` with:
- `components/` — Feature-scoped Vue SFCs.
- `store/` — Pinia store (Option API with typed state).
- `types/` — TypeScript interfaces and type exports.
- `pages/` — Feature-level pages (optional, used by auth).

Modules import from `core/` but **never from other modules**. This keeps boundaries clean.

### 2. Core Shared Layer

`src/core/` contains reusable infrastructure:
- **Base components** — `BaseCard`, `BaseInput`, `BaseCheckbox`, `BaseSelect`, `BaseTextarea`, `BaseTag`, `BaseListItem`.
- **Chart wrappers** — Thin Chart.js wrappers for activity/productivity visualization.
- **Icons** — Centralized icon exports.
- **Composables** — Shared reactive logic.
- **Utils** — Pure utility functions.

### 3. Layout-Based Routing

`src/router/index.ts` uses two top-level layout routes:
- **DashboardLayout** — wraps all authenticated pages (sidebar + content).
- **AuthLayout** — wraps guest-only pages (login).

A `beforeEach` navigation guard checks `useAuthStore().isAuthenticated`.

### 4. Pinia State Management

Stores follow the Option API pattern with typed state:
```typescript
defineStore('storeName', {
  state: (): TypedState => ({ ... }),
  getters: { ... },
  actions: { ... }
})
```

Exception: the tasks module's `todoStore.ts` uses the Composition API pattern (`defineStore('todo', () => { ... })`).

### 5. Component Composition

Pages compose feature components and core components:
- Pages handle layout, data flow, and composition.
- Feature components handle domain-specific UI.
- Core components handle reusable UI primitives.

## Known Architectural Issues

1. **No API layer** — all data is mock/in-memory. No HTTP client configured.
2. **ActivityListView reuse** — multiple routes (upcoming, categories, tags, completed, settings) share the same component, creating implicit coupling.
3. **Legacy stores** — `src/stores/todoStore.js` (plain JS) and the newer `src/modules/tasks/stores/todoStore.ts` coexist.
4. **Page naming** — `TodayPage.vue` and `TodayView.vue` both exist; only `TodayView.vue` is routed.
5. **Chart.js global registration** — `Chart.register(...registerables)` in `main.ts` loads all chart types; could be tree-shaken.
6. **No error boundary** — `src/core/error-boundaries/` exists as a directory but usage in pages is unclear.
